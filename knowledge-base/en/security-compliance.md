# Gatewise security and compliance

Gatewise is built on a Zero Trust model: no user or device is trusted by default, and
every access request is verified against identity, device posture, and policy before
access is granted.

## Authentication and access
- Multi-factor authentication is available on all plans and can be enforced by policy.
- Device posture checks (OS version, disk encryption, screen lock) gate access on Pro and Enterprise.
- Conditional access policies on Enterprise can allow, block, or step up authentication based on location, network, and risk signals.

## Encryption
- All data in transit is encrypted with TLS 1.2 or higher.
- All data at rest is encrypted with AES-256.

## Certifications and compliance
- SOC 2 Type II (audited annually).
- ISO/IEC 27001.
- GDPR compliant; a Data Processing Addendum is available on request.
- Annual third-party penetration testing; summary reports available to Enterprise customers under NDA.

## Data residency
- Enterprise customers can choose a US or EU data region. Data residency selection is not available on Starter or Pro.

## Logging and monitoring
- Audit logs record sign-ins, policy changes, and admin actions.
- Retention is 7 days (Starter), 90 days (Pro), and 1 year with SIEM export (Enterprise).

## Subprocessors
- Gatewise uses a small set of infrastructure and email subprocessors; the current list is published on the trust page and updated when it changes.

## Reporting a security issue
- Suspected vulnerabilities or account compromise should be reported to the security team immediately. Suspected data exposure is always treated as a P1 incident (see the support and escalation policy).
