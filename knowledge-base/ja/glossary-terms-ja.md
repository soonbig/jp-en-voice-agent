# Gatewise 用語集（読み・別称・英語表記）

本ドキュメントは、Gatewise の主要な用語について、日本語での読み方・カタカナ表記・英語表記・略称をまとめたものです。話し方や表記がどの形であっても、同じ機能を指していることを示します。

## シングルサインオン（SSO）
シングルサインオン、サインオン、エスエスオー、SSO、single sign-on は、すべて同じ機能を指します。一度の認証で複数のアプリケーションにログインできる仕組みです。設定方法は「SSO 設定ガイド（Okta 連携）」を参照してください。

## ログイン
ログイン、ログオン、サインイン、login は、すべて同じ操作を指します。利用者が ID プロバイダー経由で Gatewise にアクセスする操作です。

## SAML
サムル、エスエーエムエル、SAML は同じものを指します。SSO で利用する認証の標準規格（SAML 2.0）です。署名証明書の更新については「SAML 署名証明書のローテーション手順」を参照してください。

## SCIM（プロビジョニング）
スキム、エスシーアイエム、SCIM、プロビジョニングは、ユーザーの自動作成・更新・無効化を行う仕組みを指します。設定方法は「SCIM プロビジョニング設定ガイド」を参照してください。

## 多要素認証（MFA）
多要素認証、エムエフエー、MFA、二要素認証、ツーファクター認証は、パスワードに加えて追加の認証要素を求めるセキュリティ機能を指します。

## デバイスポスチャ（端末ポスチャ）
デバイスポスチャ、端末ポスチャ、device posture は、アクセス前に端末がセキュリティ要件を満たしているかを確認する機能です。設定方法は「デバイスポスチャチェック設定ガイド」を参照してください。

## アクセスポリシー
アクセスポリシー、access policy は、誰がどの条件でどのアプリケーションにアクセスできるかを定義するルールを指します。

## ID プロバイダー（IdP）
ID プロバイダー、アイディーピー、IdP は、認証を担うサービス（Okta、Microsoft Entra ID など）を指します。

## Okta / Microsoft Entra ID
オクタ（Okta）、エントラ（Entra ID、旧 Azure AD）は、Gatewise が連携できる代表的な ID プロバイダーです。

## 署名証明書
署名証明書、証明書、サーティフィケート、X.509 証明書は、SAML 認証で署名の検証に使用する証明書を指します。有効期限があり、定期的なローテーション（更新）が必要です。
