# Gatewise SCIM プロビジョニング設定ガイド

SCIM（System for Cross-domain Identity Management）を使用すると、ID プロバイダー（Okta、Microsoft Entra ID など）でのユーザーの作成・更新・削除を、Gatewise に自動的に同期できます。手動でのユーザー管理が不要になり、退職者のアクセスも自動で無効化されます。

## SCIM でできること
- ユーザーの自動作成（プロビジョニング）
- 属性（氏名・メールアドレス・部署など）の自動更新
- ユーザーの自動無効化（デプロビジョニング）— 退職者のアクセスを速やかに停止
- グループの同期

## 前提条件
- Gatewise の管理者権限
- SSO（SAML）が先に設定済みであること（「SSO 設定ガイド」を参照）
- ID プロバイダー側で SCIM プロビジョニングが利用可能なプランであること

## 手順

### 1. Gatewise で SCIM トークンを発行
1. Gatewise 管理コンソールで「設定」→「プロビジョニング」→「SCIM」を開きます。
2. 「SCIM を有効化」をクリックします。
3. 表示される **SCIM ベース URL** と **シークレットトークン** を控えます。トークンは一度しか表示されないため、安全に保管してください。

### 2. ID プロバイダー側で SCIM を設定
1. ID プロバイダー（例：Okta）のアプリケーション設定で「Provisioning」タブを開きます。
2. **SCIM Connector base URL** に Gatewise の SCIM ベース URL を入力します。
3. **API Token（Bearer Token）** に Gatewise のシークレットトークンを入力します。
4. 接続テスト（Test Connection）を実行し、成功することを確認します。

### 3. 属性マッピングと同期設定
1. ユーザーの属性マッピングを設定します。最低限、メールアドレス（userName）、姓、名をマッピングします。
2. 「ユーザーの作成」「属性の更新」「ユーザーの無効化」を有効化します。
3. 必要に応じてグループの同期（Push Groups）を設定します。

### 4. 動作確認
- ID プロバイダー側でテストユーザーを作成し、数分以内に Gatewise にユーザーが作成されることを確認します。
- ID プロバイダー側でユーザーを無効化し、Gatewise 側でもアクセスが無効化されることを確認します。

## よくある問題と対処
- **接続テストが 401 で失敗する**：シークレットトークンが正しくない、または期限切れです。Gatewise で新しいトークンを発行し直してください。
- **ユーザーが同期されない**：ID プロバイダー側でユーザーがアプリに割り当てられているか確認してください。割り当てられていないユーザーは同期されません。
- **退職者のアクセスが残る**：「ユーザーの無効化」が有効になっているか確認してください。既定では無効化のみが同期され、ユーザーの完全削除は行われません。

## 補足
- SCIM による無効化は、ID プロバイダーでの無効化操作から通常数分以内に反映されます。即時にアクセスを遮断したい場合は、Gatewise 管理コンソールから手動でセッションを失効させてください。
