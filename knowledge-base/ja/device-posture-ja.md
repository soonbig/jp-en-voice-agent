# Gatewise デバイスポスチャ（端末ポスチャ）チェック設定ガイド

デバイスポスチャチェックを使用すると、アクセスを許可する前に、接続元の端末が組織のセキュリティ要件を満たしているかを確認できます。要件を満たさない端末からのアクセスをブロックまたは制限できます。

## チェックできる項目
- OS のバージョン（最低バージョンの要求）
- ディスク暗号化の有効化（macOS の FileVault / Windows の BitLocker）
- ファイアウォールの有効化
- 管理対象端末であること（MDM 登録済み）
- スクリーンロック・パスコードの設定

## 前提条件
- Gatewise の管理者権限
- 対象端末に Gatewise デバイスエージェント、または対応する MDM が導入されていること

## 手順

### 1. ポスチャチェックを作成
1. 管理コンソールで「設定」→「デバイス」→「ポスチャチェック」を開きます。
2. 「ポスチャチェックを追加」をクリックします。
3. チェック項目（例：「ディスク暗号化が有効」「OS バージョンが最低要件以上」）を選択します。
4. 対象 OS（macOS / Windows）を指定して保存します。

### 2. アクセスポリシーに適用
1. 「設定」→「アクセスポリシー」を開きます。
2. 対象のアプリケーションまたはポリシーを選択します。
3. 条件に、作成したポスチャチェックを追加します。
4. 要件を満たさない場合の動作（「ブロック」または「制限付きアクセス」）を選択します。

### 3. 動作確認
- 要件を満たす端末からはアクセスでき、満たさない端末はブロックされることを確認します。

## OS 別の注意点
- **macOS**：ディスク暗号化チェックは FileVault の状態を参照します。デバイスエージェントにフルディスクアクセス権限の付与が必要です。
- **Windows**：BitLocker と Windows ファイアウォールの状態を参照します。

## よくある問題と対処
- **正常な端末がブロックされる**：デバイスエージェントが最新か、必要な権限（macOS のフルディスクアクセスなど）が付与されているかを確認してください。
- **チェック結果が更新されない**：ポスチャ情報は一定間隔で再評価されます。即時に再評価するには、端末でデバイスエージェントを再同期してください。
