# Gatewise SSO 設定ガイド（Okta 連携）

Gatewise は SAML 2.0 によるシングルサインオン（SSO）に対応しています。本ガイドでは、ID プロバイダー（IdP）として Okta を使用して Gatewise への SSO を設定する手順を説明します。

## 前提条件
- Gatewise の管理者（Administrator）権限
- Okta の管理者権限
- 対象ユーザーが Okta 上に登録済みであること

## 設定の流れ
1. Gatewise 管理コンソールで SAML 接続を作成する
2. Okta 側でアプリケーションを作成する
3. 両者のメタデータ（署名証明書・エンドポイント）を交換する
4. ユーザーを割り当ててテストする

## 手順

### 1. Gatewise で SAML 接続を作成
1. Gatewise 管理コンソールにログインし、「設定」→「認証」→「シングルサインオン」を開きます。
2. 「SAML 接続を追加」をクリックします。
3. 表示される **ACS URL（アサーション コンシューマー サービス URL）** と **エンティティ ID（SP Entity ID）** を控えておきます。Okta の設定で使用します。

### 2. Okta でアプリケーションを作成
1. Okta 管理画面で「Applications」→「Create App Integration」を選択します。
2. 「SAML 2.0」を選択して次へ進みます。
3. **Single sign-on URL** に Gatewise の ACS URL を入力します。
4. **Audience URI（SP Entity ID）** に Gatewise のエンティティ ID を入力します。
5. 属性マッピングで、メールアドレスを NameID として送信するように設定します。

### 3. メタデータの交換
1. Okta のアプリ設定画面から **SSO URL（IdP ログイン URL）** と **署名証明書（X.509）** を取得します。
2. Gatewise の SAML 接続設定に戻り、取得した SSO URL と署名証明書を貼り付けます。
3. 設定を保存します。

### 4. ユーザーの割り当てとテスト
1. Okta で対象ユーザーまたはグループをこのアプリケーションに割り当てます。
2. Gatewise のログイン画面で「SSO でログイン」を選択し、Okta 経由でログインできることを確認します。

## よくあるエラーと対処
- **「SAML レスポンスの署名が検証できません」**：Gatewise に登録した署名証明書が Okta 側の最新の証明書と一致していません。証明書を取得し直して登録してください。
- **「ユーザーが見つかりません」**：Okta から送信される NameID（メールアドレス）が、Gatewise 上のユーザーのメールアドレスと一致しているか確認してください。
- **ログインがループして完了しない**：ACS URL またはエンティティ ID の設定ミスが原因です。手順 1 で控えた値と一致しているか確認してください。

## 補足
- SSO の強制（パスワードログインの無効化）は、設定が正常に動作することを確認したうえで有効化することを推奨します。
- 同様の手順で Microsoft Entra ID（旧 Azure AD）とも連携できます。
