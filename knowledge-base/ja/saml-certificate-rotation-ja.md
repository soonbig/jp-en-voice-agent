# Gatewise SAML 署名証明書のローテーション手順

SAML SSO で使用する署名証明書には有効期限があります。期限切れによるログイン障害を防ぐため、定期的に証明書をローテーション（更新）する必要があります。本手順では、ダウンタイムを発生させずに証明書を更新する方法を説明します。

## ローテーションが必要なタイミング
- 証明書の有効期限が近づいたとき（Gatewise は有効期限の 30 日前に管理者へ通知します）
- 秘密鍵の漏えいが疑われるとき
- セキュリティポリシーに基づく定期更新

## 手順（無停止での更新）

### 1. 新しい証明書を準備
- ID プロバイダー（Okta / Entra ID）側で新しい署名証明書を生成します。多くの ID プロバイダーでは、既存の証明書と並行して新しい証明書を追加できます。

### 2. Gatewise に新しい証明書を登録
1. 管理コンソールで「設定」→「認証」→「シングルサインオン」を開きます。
2. 対象の SAML 接続を選択し、「署名証明書」の項目で新しい証明書（X.509）を追加します。
3. Gatewise は複数の証明書を同時に信頼できるため、この時点では既存のログインに影響はありません。

### 3. ID プロバイダー側で署名証明書を切り替え
- ID プロバイダーの設定で、SAML レスポンスの署名に使用する証明書を新しいものに切り替えます。

### 4. 動作確認と旧証明書の削除
1. SSO でログインできることを確認します。
2. 問題がなければ、Gatewise から古い証明書を削除します。

## よくある問題と対処
- **切り替え後にログインできない**：ID プロバイダーが新しい証明書で署名しているのに、Gatewise にその証明書が登録されていない可能性があります。手順 2 を再確認してください。
- **「署名を検証できません」エラー**：ID プロバイダー側で署名証明書の切り替えが完了していない、または別の証明書が使用されています。

## 補足
- 証明書の有効期限は、管理コンソールの「シングルサインオン」画面でいつでも確認できます。
